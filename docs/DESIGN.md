# Design System: 28-github-desktop-tutorial

## Visual Philosophy
The design follows an **Engineering Blueprint / GitFlow Technical** aesthetic. It uses a "Slate Deep-Space" foundation with "GitFlow Orange" (#f05032) technical accents, focusing on schematic clarity, path-finding motion, and protocol transparency.

## Design Patterns
-   **Schematic Nodes**: Heavy slate containers (Slate-800) with internal orange borders and monospaced "NODE_ID" tracking.
-   **Blueprint Background**: Specialized CSS-grid pattern reminiscent of technical plotting paper, rendered with high transparency.
-   **Git Terminal**: Monospaced, high-contrast CLI briefings with syntax highlighting and Git-orange command tokens.
-   **Path Navigation**: Directional icons (ArrowRight) that pulse when nodes are active to signify the "Distributed Pipeline" flow.

## Color Palette
-   **Slate Foundation**: Primary Background (#0f172a)
-   **GitFlow Orange**: Primary Action & Protocol (#f05032)
-   **Protocol Emerald**: Verification & Security (#10b981)
-   **Terminal Slate**: Code Background (#020617)

## Interaction Design
-   **Schmatic Selection**: 300ms scale-motion with orange ring-glow to signify an active stage in the Git lifecycle.
-   **Protocol Pulse**: Continuous opacity iteration on active path arrows to guide the user through the pipeline.
-   **Code Kinetic**: 500ms slide-fade transitions for terminal briefings when switching nodes.
